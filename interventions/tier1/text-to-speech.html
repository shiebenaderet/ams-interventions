What It Is

Technology that reads text aloud to students (TTS) or converts spoken words to written text (STT).

What Quality Implementation Looks Like

‚úÖ In the Classroom (What TO Do):

  • Students know how to activate these tools independently
  • These tools are normalized (not stigmatized as "only for struggling students")
  • Teacher uses TTS/STT in front of class occasionally
  • Students choose when to use based on task demands
  • Works seamlessly with Canvas, Google Docs, PDFs

‚ùå Red Flags (What NOT to Do):

  • "The technology is available but students don't know how to use it"
  • Only special ed students allowed to use it
  • Students have to ask permission each time
  • Technical problems go unsolved
  • No instructional time spent teaching the tool

Implementation Steps

Initial Setup (Beginning of Year)

Whole-class tech lesson (20-30 min):

  • "Everyone will learn this tool because everyone can benefit"
  • Demonstrate on your own screen
  • Students practice on low-stakes text (announcements, fun article)
  • Troubleshoot common issues

Create visual guide:

  • Step-by-step screenshots
  • Post on Canvas and on wall
  • Include troubleshooting tips

Practice scenarios:

  • "Today's reading is challenging. I'm going to use text-to-speech to help me."
  • Give students choice: "You may read silently, with a partner, or use TTS"

Ongoing Use

  • For assessments: "All students may use TTS/STT on this test"
  • For assignments: Build in expectation ("Use TTS for at least one source in your research")
  • Normalize: "I use speech-to-text for my emails all the time"

Monitoring Effectiveness

  • Survey students: Which tool do you use? When? Is it helpful?
  • Track: Do students who use TTS/STT show better comprehension?
  • Adjust: If students aren't using it, why not? (Don't know how? Embarrassed? Doesn't help?)
